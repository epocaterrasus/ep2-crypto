Order Book Imbalance (OBI)\n\n### Mathematical Formulas\n\n**Basic (Level 1) OBI:**\n\n```\nOBI = (V_bid - V_ask) / (V_bid + V_ask)\n```\n\nwhere `V_bid` = total volume at best bid, `V_ask` = total volume at best ask. VWAP Variants for Order Books\n\n### Standard VWAP (Trade-Based)\n\n```\nVWAP = Σ(P_i * Q_i) / Σ(Q_i)\n```\n\nwhere the sum is over all trades in a time window.\n\n### Micro-VWAP from L2 Order Book (Volume-Weighted Mid-Price)\n\nThis computes a theoretical execution price for a given quantity from the current order book state:\n\n**Ask-side micro-VWAP (cost to buy Q units):**\n\n```\nμVWAP_ask(Q) = Σ(i=1..k) P_ask_i * min(V_ask_i, Q_remaining_i) / Q\n```\n\nwhere `Q_remaining_i = Q - Order Flow Imbalance (OFI) — Cont, Stoikov & Talreja (2014)\n\n### The Original CST Model\n\n**Paper**: \"The Price Impact of Order Book Events\" (Quantitative Finance, 2014)\n\nThe key insight: track changes in the order book state at the best bid/ask between consecutive observations.\n\n**Event-level OFI definition:**\n\nAt each order book update n, define:\n\n```\ne_n = 1_{P_bid_n ≥ P_bid_{n-1}} * V_bid_n - 1_{P_bid_n ≤ P_bid_{n-1}} * V_bid_{n-1}\n     - 1_{P_ask_n ≤ P_ask_{n-1}} * V_ask_n + 1_{P_ask_n ≥ P_ask_{n-1}} * V_ask_{n-1}\n```\n\nMore explicitly, the four cases for each side:\n\n**Bid side contribution:**\n- If `P_bid_n > P_bid_{n-1}`: new best bid appeared → +V_bid_n\n- If `P_bid_n = P_bid_{n-1}`: volume change at same price → +(V_bid_n - V_bid_{n-1})\n- If `P_bid_n < P_bid_{n-1}`: best bid was consumed → -V_bid_{n-1}\n\n**Ask side contribution:**\n- If `P_ask_n < P_ask_{n-1}`: new best ask appeared → -V_ask_n\n- If `P_ask_n = P_ask_{n-1}`: volume change at same price → -(V_ask_n - V_ask_{n-1})\n- If `P_ask_n > P_ask_{n-1}`: best ask was consumed → +V_ask_{n-1}\n\n**Aggregated OFI over interval [t, t+Δt]:**\n\n```\nOFI_t = Σ(n: events in [t, t+Δt]) e_n\n```\n\n**The linear price impact model:**\n\n```\nΔP_t = α + β * OFI_t + ε_t\n```\n\nwhere β (the price impact coefficient) is estimated via OLS. n_buckets:i]\n        vpin = np.sum(window) / (n_buckets * bucket_volume)\n        vpin_series.append(vpin)\n    \n    return np.array(vpin_series)\n```\n\n### Crypto-Specific Results\n\n- **Abad & Yague (2012)** adapted VPIN to FX. Kyle's Lambda (Price Impact Coefficient)\n\n### Background\n\n**Original paper**: Kyle (1985), \"Continuous Auctions and Insider Trading\" (Econometrica).\n\nKyle's Lambda (λ) measures how much price moves per unit of net order flow. High λ → illiquid market / high information asymmetry.\n\n### The Formula\n\n**Basic estimation via linear regression:**\n\n```\nΔP_t = α + λ * OF_t + ε_t\n```\n\nwhere:\n- `ΔP_t` = price change in interval t (often mid-price change)\n- `OF_t` = signed net order flow = Σ(trades in t) sign_i * volume_i\n- `sign_i` = +1 for buyer-initiated, -1 for seller-initiated (classified via Lee-Ready or similar)\n\nλ is estimated via OLS. Units: price per unit volume (e.g., $/BTC).\n\n**Square-root variant (more robust for large orders):**\n\n```\nΔP_t = α + λ_sqrt * sign(OF_t) * sqrt(|OF_t|) + ε_t\n```\n\nThis follows the empirical finding that price impact is concave in volume (Bouchaud, 2010).\n\n**Rolling estimation for time-varying λ:**\n\nUse a rolling window of W intervals (e.g., W=100 one-minute bars):\n\n```\nλ_t = Cov(ΔP, OF)[t-W:t] / Var(OF)[t-W:t]\n```\n\n### Python Pseudocode\n\n```python\ndef classify_trades_lee_ready(trades_df, quotes_df):\n    \"\"\"Lee-Ready classification: compare trade price to mid-quote.\"\"\"\n    # Merge each trade with the most recent quote\n    merged = trades_df.merge_asof(quotes_df, on='timestamp')\n    mid = (merged['bid'] + merged['ask']) / 2\n    \n    signs = np.where(merged['price'] > mid, 1,\n            np.where(merged['price'] < mid, -1, 0))\n    # Tick rule for trades at mid\n    for i in range(1, len(signs)):\n        if signs[i] == 0:\n            signs[i] = signs[i - 1]\n    \n    merged['sign'] = signs\n    return merged\n\n\ndef compute_kyles_lambda(price_changes: np.ndarray, \n                         signed_volume: np.ndarray,\n                         window: int = 100) -> np.ndarray:\n    \"\"\"Rolling Kyle's lambda estimation.\"\"\"\n    lambdas = np.full(len(price_changes), np.nan)\n    \n    for i in range(window, len(price_changes)):\n        dp = price_changes[i - window:i]\n        of = signed_volume[i - window:i]\n        \n        var_of = np.var(of)\n        if var_of > 0:\n            lambdas[i] = np.cov(dp, of)[0, 1] / var_of\n    \n    return lambdas\n```\n\n### Crypto Results & Relevance\n\n- **Donier & Bonart (2015)**: Estimated λ for Bitcoin on early exchanges. Order Book Resilience\n\n### Definition & Measurement\n\nOrder book resilience measures how quickly the order book recovers its depth after a large trade or shock. Spoofing/Layering Detection from L2 Data\n\n### What It Is\n\nSpoofing: placing large orders with intent to cancel before execution, to create false impression of supply/demand. Layering: multiple spoof orders at different price levels.\n\n### Detection Methods\n\n**Method 1: Order-to-Trade Ratio (OTR) at price level**\n\n```\nOTR_level = Volume_placed_and_cancelled / Volume_actually_traded\n```\n\nHigh OTR at specific levels (>10x) is suspicious.\n\n**Method 2: Asymmetric fleeting orders (Cao, Chen & Hansch, 2008 methodology adapted)**\n\nTrack orders that:\n1. Are large (>90th percentile of order size)\n2. Appear on one side of the book\n3. Are cancelled within T seconds (T < 2s for crypto)\n4. Coincide with trades executing on the opposite side\n\n```\nSpoof_score = Σ(cancelled large orders on side S within T) / Σ(all large orders on side S)\n```\n\nwhere S is the side opposite to where trades are executing.\n\n**Method 3: Book pressure flipping**\n\n```\nBP_t = (ΔV_bid_placed - ΔV_bid_cancelled) - (ΔV_ask_placed - ΔV_ask_cancelled)\n```\n\nIf BP_t is strongly positive (buy pressure via large bids placed) but actual trades are net selling, this is a spoofing signature.\n\n**Method 4: Snapshot-based detection (when only L2 snapshots available)**\n\nCompare consecutive snapshots:\n```\nPhantom_volume_bid = max(0, V_bid_t - V_bid_{t+1})  when no trades occurred at that level\n```\n\nLarge phantom volume appearing and disappearing without any trades at that level indicates cancelled (potentially spoof) orders.\n\n### Python Pseudocode\n\n```python\ndef detect_spoof_from_snapshots(snapshots: list, \n                                 trades: list,\n                                 size_threshold_pct: float = 90,\n                                 lifetime_threshold_s: float = 2.0):\n    \"\"\"\n    Detect potential spoofing from L2 snapshot diffs.\n    snapshots: list of (timestamp, bids_dict, asks_dict)\n    \"\"\"\n    suspicious_events = []\n    \n    for i in range(1, len(snapshots)):\n        t_prev, bids_prev, asks_prev = snapshots[i - 1]\n        t_curr, bids_curr, asks_curr = snapshots[i]\n        dt = t_curr - t_prev\n        \n        if dt > lifetime_threshold_s:\n            continue\n        \n        # Find large volumes that appeared then disappeared\n        for price in set(bids_prev.keys()) | set(bids_curr.keys()):\n            vol_prev = bids_prev.get(price, 0)\n            vol_curr = bids_curr.get(price, 0)\n            \n            # Large order appeared in previous snapshot, gone now\n            if vol_prev > vol_curr and (vol_prev - vol_curr) > size_threshold:\n                # Check if any trades occurred at this price\n                trades_at_price = [t for t in trades \n                                   if t_prev <= t['time'] <= t_curr \n                                   and t['price'] == price]\n                if not trades_at_price:\n                    suspicious_events.append({\n                        'time': t_curr,\n                        'side': 'bid',\n                        'price': price,\n                        'phantom_volume': vol_prev - vol_curr\n                    })\n    \n    return suspicious_events\n```\n\n### Relevance to Crypto\n\n- **Spoofing is rampant in crypto** due to less regulation. The CFTC has brought cases against crypto spoofers but enforcement is limited.\n- **Aitken, Cumming & Zhan (2015)**: Found spoofing detection signals are predictive of short-term reversals: price moves induced by spoofing tend to reverse within 30-120 seconds.\n- **Practical value for prediction**: Detecting potential spoofing allows you to:\n  1. **Fade the move**: If buy-side spoofing detected (large phantom bids), expect downward pressure (the spoofer wants to sell)\n  2. **Filter OBI signals**: If OBI is positive but driven by fleeting phantom volume, discount it\n- **Limitation from snapshots**: With 100ms-1s L2 snapshots, you may miss very fast spoofs (<100ms). Full L3 data (individual order tracking) is ideal but rarely available for crypto.\n- **Signal quality**: Moderate. Spoofing detection is noisy — many false positives from legitimate market makers adjusting quotes. Best used as a **filter** on other signals rather than standalone.\n\n### Data & Complexity\n\n- **Data**: Ideally L3/order-by-order data. No ground truth available.\n\n---\n\n## 8. Absorption Patterns\n\n### Definition\n\nAbsorption occurs when large resting limit orders absorb aggressive market orders without the price moving. This indicates strong conviction by the passive side and often precedes a reversal (the aggressive side runs out of steam).\n\n### Detection Methods\n\n**Method 1: Volume-at-price absorption ratio**\n\n```\nAbsorption_ratio = Volume_traded_at_level / Volume_remaining_at_level\n```\n\nIf `Volume_traded_at_level` is large (many aggressive orders hitting the level) but the level holds (volume replenishes or price doesn't break through):\n\n```\nAbsorbed = True  if  Σ(trades at price P in window) > k * V_resting_at_P  AND  P still best bid/ask\n```\n\nTypical k ∈ [2, 5]: the level absorbed 2-5x its visible depth.\n\n**Method 2: Delta-volume divergence**\n\nTrack cumulative delta (net aggressive buying) vs. price:\n\n```\nCumDelta_t = Σ(buy_volume - sell_volume) from t0 to t\n```\n\nIf CumDelta is strongly positive (net buying) but price is flat or declining → absorption by sell-side.\n\n**Method 3: Iceberg detection from snapshots**\n\nIcebergs are large hidden orders that display only a small visible portion. Detect from snapshots:\n\n```\nIceberg_suspected = True  if:\n  1. Volume at level P decreases by V_trade (from trade data)\n  2. But next snapshot shows volume at P replenished by ≥ V_trade\n  3. No new visible order placement detected\n  Repeated n ≥ 3 times at same price level\n```\n\n### Python Pseudocode\n\n```python\ndef detect_absorption(trades: list, snapshots: list, \n                      window_s: float = 30.0,\n                      absorption_multiplier: float = 2.0):\n    \"\"\"\n    Detect absorption at price levels.\n    \"\"\"\n    events = []\n    \n    for i, snap in enumerate(snapshots):\n        t = snap['timestamp']\n        \n        # Look at best bid and best ask\n        for side, price_key, vol_key in [\n            ('bid', 'best_bid', 'bid_vol'),\n            ('ask', 'best_ask', 'ask_vol')\n        ]:\n            price = snap[price_key]\n            visible_vol = snap[vol_key]\n            \n            # Count trades at this price in the window\n            trades_at_price = [tr for tr in trades \n                               if t - window_s <= tr['time'] <= t\n                               and abs(tr['price'] - price) < 1e-8]\n            total_traded = sum(tr['volume'] for tr in trades_at_price)\n            \n            # Absorption: lots traded but level still holds\n            if total_traded > absorption_multiplier * visible_vol and visible_vol > 0:\n                events.append({\n                    'time': t,\n                    'side': side,\n                    'price': price,\n                    'volume_absorbed': total_traded,\n                    'visible_volume': visible_vol,\n                    'ratio': total_traded / visible_vol\n                })\n    \n    return events\n\n\ndef detect_iceberg(snapshots: list, trades: list, \n                   min_refills: int = 3):\n    \"\"\"Detect iceberg orders from repeated refills at same price.\"\"\"\n    refill_counts = {}  # (side, price) -> count\n    \n    for i in range(1, len(snapshots)):\n        prev, curr = snapshots[i - 1], snapshots[i]\n        t_prev, t_curr = prev['timestamp'], curr['timestamp']\n        \n        for side in ['bid', 'ask']:\n            price = prev[f'best_{side}']\n            if curr[f'best_{side}'] != price:\n                continue  # Level changed\n            \n            vol_prev = prev[f'{side}_vol']\n            vol_curr = curr[f'{side}_vol']\n            \n            # Volume decreased (trade) then appeared to refill\n            trades_between = sum(\n                tr['volume'] for tr in trades\n                if t_prev <= tr['time'] <= t_curr\n                and abs(tr['price'] - Microprice (Gatheral & Stoikov)\n\n### The Formula\n\n**Paper**: Gatheral & Oomen (2010); refined by Stoikov (2018), \"The Micro-Price: A High-Frequency Estimator of Future Prices\"\n\n**Standard microprice:**\n\n```\nMicroprice = P_ask * (V_bid / (V_bid + V_ask)) + P_bid * (V_ask / (V_bid + V_ask))\n```\n\nIntuition: if bid volume >> ask volume, the microprice is pulled toward the ask (buying pressure likely to push price up). This is volume-weighted interpolation between best bid and best ask.\n\nEquivalently:\n\n```\nMicroprice = P_mid + (V_bid - V_ask) / (2 * (V_bid + V_ask)) * Spread\n```\n\nwhere `P_mid = (P_bid + P_ask) / 2` and `Spread = P_ask - P_bid`.\n\n**Stoikov's conditional microprice (2018):**\n\nThe full model conditions on the **imbalance** I = V_bid / (V_bid + V_ask) ∈ [0, 1]:\n\n```\nMicroprice(I) = E[P_mid(t + Δt) | I_t = I]\n```\n\nEstimated non-parametrically by bucketing imbalance values and computing the average future mid-price for each bucket. The resulting function is typically sigmoid-shaped:\n\n```\nMicroprice(I) ≈ P_mid + a * (2I - 1) + b * (2I - 1)^3 + ...\n```\n\nIn practice, a linear approximation `Microprice ≈ P_mid + α * (I - 0.5) * Spread` with α calibrated from data works well. The standard formula (above) corresponds to α = 1.\n\n**Multi-level microprice (extension):**\n\n```\nI_k = V_bid_k / (V_bid_k + V_ask_k)    for level k\nMicroprice_multi = P_mid + Σ(k=1..K) α_k * (I_k - 0.5) * spread_k\n```\n\nwhere α_k are calibrated coefficients (typically decreasing with k).\n\n### Python Pseudocode\n\n```python\ndef microprice(best_bid: float, best_ask: float, \n               bid_vol: float, ask_vol: float) -> float:\n    \"\"\"Standard Gatheral-Stoikov microprice.\"\"\"\n    total_vol = bid_vol + ask_vol\n    if total_vol == 0:\n        return (best_bid + best_ask) / 2\n    \n    imbalance = bid_vol / total_vol\n    return best_ask * imbalance + best_bid * (1 - imbalance)\n\n\ndef conditional_microprice(mid_prices: np.ndarray, \n                           imbalances: np.ndarray,\n                           horizon: int = 1,\n                           n_bins: int = 20):\n    \"\"\"\n    Calibrate Stoikov's conditional microprice.\n    mid_prices: time series of mid-prices\n    imbalances: time series of I = V_bid / (V_bid + V_ask)\n    horizon: prediction horizon in ticks\n    \"\"\"\n    future_mids = np.roll(mid_prices, -horizon)\n    future_changes = future_mids - mid_prices\n    \n    # Remove last 'horizon' entries\n    imb = imbalances[:-horizon]\n    changes = future_changes[:-horizon]\n    \n    # Bin by imbalance\n    bins = np.linspace(0, 1, n_bins + 1)\n    bin_centers = []\n    bin_means = []\n    \n    for i in range(n_bins):\n        mask = (imb >= bins[i]) & (imb < bins[i + 1])\n        if np.sum(mask) > 30:  # minimum sample\n            bin_centers.append((bins[i] + bins[i + 1]) / 2)\n            bin_means.append(np.mean(changes[mask]))\n    \n    # Fit polynomial (typically cubic)\n    coeffs = np.polyfit(\n        np.array(bin_centers) - 0.5,  # center at 0\n        bin_means, \n        deg=3\n    )\n    \n    return coeffs  # Use np.polyval(coeffs, I - 0.5) for prediction\n```\n\n### Crypto-Specific Results\n\n- **Stoikov (2018)**: Original paper used equity data.
