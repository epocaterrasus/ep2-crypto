Kelly Criterion with Uncertain Probabilities\n\n### Standard Kelly\n\nFor a binary bet with probability `p` of winning, odds `b`:\n```\nf* = (bp - q) / b = p - q/b\n```\nwhere `q = 1-p`. For equal odds (b=1): `f* = 2p - 1`.\n\nFor continuous returns with estimated mean `μ` and variance `σ²`:\n```\nf* = μ / σ²\n```\n\n### The Problem with Estimated Probabilities\n\nIf `p` is estimated with error, Kelly overbets catastrophically. If true p = 0.52 but you estimate p = 0.58, Kelly says bet 16% instead of 4%. The expected log-growth becomes **negative** with estimation error.\n\n### Adjusted Kelly Formulas\n\n**1. Fractional Kelly:**\n```\nf_adjusted = k * f_kelly    where k ∈ (0, 1)\n```\nCommon choices:\n- Half-Kelly (k=0.5): Most common in practice. Sacrifices ~25% of growth rate but dramatically reduces variance and max drawdown.\n- Quarter-Kelly (k=0.25): Very conservative, for when probability estimates are noisy (as in financial models).\n\n**Theoretical justification** (MacLean et al., 2010): If probability estimate has standard error `σ_p`, the optimal fraction is approximately:\n```\nk ≈ 1 / (1 + (σ_p / (p - 0.5))^2)\n```\nWhen `σ_p` is large relative to edge `(p - 0.5)`, k shrinks dramatically.\n\n**2. Confidence-Weighted Kelly:**\n```python\ndef adjusted_kelly(estimated_prob, confidence, base_odds=1.0):\n    \"\"\"\n    estimated_prob: model's P(win)\n    confidence: 0-1 measure of how much to trust the probability\n        (from Venn-Abers interval width, ensemble agreement, etc.)\n    base_odds: risk/reward ratio\n    \"\"\"\n    edge = estimated_prob - (1 - estimated_prob) / base_odds\n    if edge <= 0:\n        return 0.0\n\n    kelly_fraction = edge / base_odds\n\n    # Shrink toward zero based on confidence\n    # confidence=1 -> full kelly; confidence=0 -> no bet\n    adjusted = kelly_fraction * confidence\n\n    # Additional safety: cap at quarter-kelly\n    max_fraction = kelly_fraction * 0.25\n    return min(adjusted, max_fraction)\n```\n\n**3. Bayesian Kelly (most rigorous):**\n\nInstead of a point estimate of `p`, use the posterior distribution `P(p|data)`:\n```\nf* = argmax_f E_p[log(1 + f * (p * b - q))]\n```\nThis integrates over uncertainty in `p`. With a Beta posterior:\n```\nP(p|data) = Beta(α + wins, β + losses)\n```\nThe optimal fraction can be computed numerically.\n\n**4. Abstention/Rejection in Classification\n\n### Academic Background\n\n**Classification with a reject option** (Chow, 1970): The optimal decision rule with rejection cost `c` is:\n```\nreject if max_k P(k|x) < 1 - c\n```\nWhen the cost of rejection (missing a trade) is low relative to the cost of misclassification (losing trade), you should reject aggressively.\n\n### The Chow Rule for Trading\n\n```\nTrade if: max(P(up|x), P(down|x)) > 1 - c_reject/c_error\n```\n\nWhere:\n- `c_reject` = opportunity cost of not trading ≈ 0 (there's always another trade)\n- `c_error` = cost of a wrong trade = spread + fees + adverse movement\n\nSince `c_reject / c_error` is small, the threshold should be high.\n\n### Training a Model with \"Don't Know\" Output\n\n**Three-class approach:**\n```python\n# Relabel: if |actual_return| < cost_threshold, label = \"neutral\"\n# Otherwise label = \"up\" or \"down\"\ndef create_three_class_labels(returns, cost_bps=15):\n    cost = cost_bps / 10000\n    labels = np.where(\n        returns > cost, 2,          # up (profitable long)\n        np.where(returns < -cost, 0, 1)  # down / neutral\n    )\n    return labels\n```\n\nThis explicitly teaches the model that some periods are untradeable.\n\n### Selective Prediction (Geifman & El-Yaniv, 2017)\n\nTrain a **selection function** `g(x) ∈ {0, 1}` alongside the predictor:\n```\nLoss = E[l(f(x), y) * g(x)] + λ * E[1 - g(x)]\n```\nThe first term penalizes errors on selected samples. The second penalizes rejection rate. `λ` controls the tradeoff.\n\n### SelectiveNet Architecture\n\n```python\nclass SelectiveNet(nn.Module):\n    \"\"\"\n    Neural network with learned abstention.\n    Three heads: prediction, selection, auxiliary.\n    \"\"\"\n    def __init__(self, input_dim, hidden_dim=64):\n        super().__init__()\n        self.backbone = nn.Sequential(\n            nn.Linear(input_dim, hidden_dim),\n            nn.ReLU(),\n            nn.Linear(hidden_dim, hidden_dim),\n            nn.ReLU(),\n        )\n        self.prediction_head = nn.Linear(hidden_dim, 2)\n        self.selection_head = nn.Sequential(\n            nn.Linear(hidden_dim, 1),\n            nn.Sigmoid()  # outputs selection probability\n        )\n\n    def forward(self, x):\n        features = self.backbone(x)\n        prediction = self.prediction_head(features)\n        selection = self.selection_head(features)\n        return prediction, selection\n```\n\n### Evidence on Financial Data\n\n- Ziyin et al. Drawdown-Based Gating\n\n### The Core Tension\n\nShutting down after drawdown protects capital but guarantees you miss the recovery.
