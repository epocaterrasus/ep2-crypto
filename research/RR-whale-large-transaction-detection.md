Address Clustering Techniques\n\n### How Chainalysis/Glassnode Do It\n\n**Core heuristics:**\n\n1. **Common Input Ownership Heuristic (CIOH)**: If two addresses appear as inputs in the same transaction, they are controlled by the same entity. Heuristics to identify the change address:\n   - The change output uses the same address type as the inputs\n   - The change output is a \"round number\" vs. the payment\n   - The change output is to a new address not seen before\n   - Address reuse patterns\n\n3. **Temporal Analysis**: Transactions from the same entity tend to have temporal patterns (similar times of day, consistent fee rates).\n\n4. **Behavioral Fingerprinting**: Exchanges have distinctive transaction patterns:\n   - High fan-out transactions (one input, many outputs) = exchange withdrawal batching\n   - High fan-in transactions (many inputs, one output) = UTXO consolidation\n   - Consistent fee estimation algorithms\n\n5. **Known Address Seeding**: Start with a few known exchange addresses (from public disclosures, deposit testing, community intelligence) and expand using the above heuristics.\n\n6. **Cross-chain Analysis**: If an entity operates on multiple chains, cross-referencing timing and amounts can link addresses.\n\n### Can You Do a Simplified Version?\n\n**Yes, with significant limitations:**\n\n**Minimal viable clustering approach:**\n1. Use Union-Find (disjoint set) data structure to maintain clusters efficiently.\n4. Seed known exchange addresses from public lists.\n5. Any address that clusters with a known exchange address is labeled as that exchange.\n\n**Scale challenge**: Bitcoin has ~1 billion+ addresses and ~900M+ transactions.
