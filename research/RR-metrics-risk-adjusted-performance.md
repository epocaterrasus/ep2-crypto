The corrected Sharpe:\n\n```\nSR_corrected = SR_annual × sqrt( q / (q + 2 × sum_{k=1}^{q-1} (q-k) × rho_k) )\n```\n\nWhere q = N (number of bars per year) and rho_k is the k-th autocorrelation of returns.\n\nIn practice, truncate the sum at a reasonable lag (e.g., 50-100 lags for 5-min data, covering ~4-8 hours).\n\nA simplified version using only the first-order autocorrelation:\n\n```\nSR_corrected ≈ SR_annual × sqrt(1 / (1 + 2*rho_1))\n```\n\nIf rho_1 > 0 (momentum), this shrinks the Sharpe. Sortino Ratio\n\n### Why Better Than Sharpe for Crypto\n\nSharpe penalizes upside volatility equally with downside volatility. For a crypto prediction strategy that has large positive outliers (right-skewed winners), Sharpe is too harsh. Sortino only penalizes downside deviation.\n\n### Formula\n\n```\nSortino = (mean(R) - MAR) / DD\n\nDD = sqrt(mean(min(R - MAR, 0)^2))\n```\n\nWhere MAR = Minimum Acceptable Return (often 0, or the risk-free rate per bar).\n\nDD is the **downside deviation** — the standard deviation computed only from returns below the MAR, but using the full sample size in the denominator (not just the count of negative returns). This is a key detail many implementations get wrong.\n\n### Annualization\n\n```\nSortino_annual = Sortino_per_bar × sqrt(N)\n```\n\nSame sqrt(N) as Sharpe, where N = 105,120.\n\n### Interpretation\n\n| Annualized Sortino | Quality |\n|---|---|\n| < 1.0 | Poor |\n| 1.0 - 2.0 | Acceptable |\n| 2.0 - 3.0 | Good |\n| > 3.0 | Excellent |\n\nSortino is typically higher than Sharpe for right-skewed strategies.\n\n### Sample Size\n\nSimilar to Sharpe — minimum ~50K-100K bars (6-12 months of 5-min data). The downside deviation estimate converges more slowly because it uses only a subset of returns.\n\n### Python Implementation\n\n```python\ndef sortino_ratio(\n    returns: np.ndarray,\n    mar_annual: float = 0.0,\n    annualize: bool = True,\n) -> float:\n    \"\"\"Sortino ratio using downside deviation.\"\"\"\n    mar_bar = (1 + mar_annual) ** (1 / BARS_PER_YEAR) - 1\n    excess = returns - mar_bar\n\n    # Downside deviation: use ALL observations in denominator\n    downside = np.minimum(excess, 0.0)\n    downside_dev = np.sqrt(np.mean(downside**2))\n\n    if downside_dev == 0:\n        return np.inf if np.mean(excess) > 0 else 0.0\n\n    sortino = np.mean(excess) / downside_dev\n    if annualize:\n        sortino *= SQRT_BARS\n    return sortino\n```\n\n---\n\n## 3. Omega Ratio\n\n### Why It Matters\n\nOmega captures the **entire return distribution**, not just mean and variance. It is the probability-weighted ratio of gains to losses relative to a threshold. It subsumes Sharpe, Sortino, and all higher moments.\n\n### Formula\n\n```\nOmega(L) = integral_{L}^{+inf} (1 - F(r)) dr  /  integral_{-inf}^{L} F(r) dr\n```\n\nWhere F is the CDF of returns and L is the threshold (often 0 or Rf).\n\nEquivalent discrete formula:\n\n```\nOmega(L) = sum(max(R_i - L, 0)) / sum(max(L - R_i, 0))\n```\n\nOr equivalently:\n\n```\nOmega(L) = 1 + (mean(R) - L) / E[max(L - R, 0)]\n```\n\n### Interpretation\n\n| Omega | Meaning |\n|---|---|\n| < 1.0 | Expected losses exceed gains (bad) |\n| = 1.0 | Break even |\n| 1.0 - 2.0 | Positive but modest |\n| > 2.0 | Strong edge |\n\nOmega is not typically annualized. Compute it on the per-bar returns directly.\n\n### Sample Size\n\nRequires enough data to estimate the tails well. At least 50K-100K bars. Sensitive to outliers in both tails.\n\n### Python Implementation\n\n```python\ndef omega_ratio(\n    returns: np.ndarray,\n    threshold: float = 0.0,\n) -> float:\n    \"\"\"Omega ratio: gains above threshold / losses below threshold.\"\"\"\n    excess = returns - threshold\n    gains = np.sum(excess[excess > 0])\n    losses = -np.sum(excess[excess < 0])\n\n    if losses == 0:\n        return np.inf if gains > 0 else 1.0\n    return gains / losses\n```\n\n---\n\n## 5. The alpha-quantile of the empirical return distribution.\n\n```\nVaR_alpha = -quantile(R, alpha)\n```\n\nConvention: VaR is reported as a positive number representing a loss.\n\n### Parametric (Gaussian) VaR\n\n```\nVaR_alpha = -(mu + z_alpha × sigma)\n```\n\nWhere z_alpha = norm.ppf(alpha). For alpha=0.01, z = -2.326.\n\nThis underestimates tail risk for crypto due to fat tails.\n\n### Cornish-Fisher VaR\n\nAdjusts the z-score for skewness and kurtosis:\n\n```\nz_cf = z + (z^2 - 1)/6 × S + (z^3 - 3z)/24 × K - (2z^3 - 5z)/36 × S^2\n```\n\nWhere S = skewness, K = excess kurtosis, z = normal quantile.\n\nThen:\n\n```\nVaR_cf = -(mu + z_cf × sigma)\n```\n\nThis is the most appropriate parametric VaR for crypto.\n\n### Which to Use?\n\nFor 5-min crypto:\n1. **Historical VaR** — best if you have enough data (>50K bars). Non-parametric, captures the actual distribution.\n2. **Cornish-Fisher VaR** — best parametric approach. Accounts for fat tails.\n3. **Gaussian VaR** — avoid for crypto. Systematically underestimates risk.\n\n### Scaling VaR Across Timeframes\n\nTo convert 5-min VaR to 1-day VaR (assuming independence):\n\n```\nVaR_1day = VaR_5min × sqrt(288)\n```\n\nThis is approximate. For more accuracy, compute VaR directly from daily returns.\n\n### Python Implementation\n\n```python\ndef var_historical(returns: np.ndarray, alpha: float = 0.05) -> float:\n    \"\"\"Historical VaR at the alpha confidence level.\"\"\"\n    return -np.percentile(returns, alpha * 100)\n\n\ndef var_parametric(returns: np.ndarray, alpha: float = 0.05) -> float:\n    \"\"\"Gaussian parametric VaR.\"\"\"\n    mu = np.mean(returns)\n    sigma = np.std(returns, ddof=1)\n    z = stats.norm.ppf(alpha)\n    return -(mu + z * sigma)\n\n\ndef var_cornish_fisher(returns: np.ndarray, alpha: float = 0.05) -> float:\n    \"\"\"Cornish-Fisher VaR adjusting for skewness and kurtosis.\"\"\"\n    mu = np.mean(returns)\n    sigma = np.std(returns, ddof=1)\n    s = stats.skew(returns)\n    k = stats.kurtosis(returns)  # excess kurtosis\n    z = stats.norm.ppf(alpha)\n\n    z_cf = (\n        z\n        + (z**2 - 1) / 6 * s\n        + (z**3 - 3 * z) / 24 * k\n        - (2 * z**3 - 5 * z) / 36 * s**2\n    )\n\n    return -(mu + z_cf * sigma)\n```\n\n---\n\n## 7. Expected Shortfall (CVaR)\n\n### Formula\n\nCVaR (Conditional Value at Risk) = the expected loss given that the loss exceeds VaR.\n\n```\nCVaR_alpha = -E[R | R <= -VaR_alpha]\n          = -(1/alpha) × integral_{-inf}^{quantile(alpha)} r × f(r) dr\n```\n\nDiscrete:\n\n```\nCVaR_alpha = -mean(R[R <= quantile(R, alpha)])\n```\n\n### Why Better Than VaR\n\n- VaR only tells you the threshold; CVaR tells you how bad it gets beyond that threshold.\n- CVaR is a **coherent risk measure** (subadditive); VaR is not.\n- Essential for crypto where tail events can be 5-10x the VaR.\n\n### Regime-Conditional CVaR\n\nCompute CVaR separately for different market regimes (high volatility, low volatility, trending, mean-reverting). Tail Ratio\n\n### Formula\n\n```\nTail Ratio = |percentile(R, 95)| / |percentile(R, 5)|\n```\n\n### Interpretation\n\n| Tail Ratio | Meaning |\n|---|---|\n| > 1.0 | Right tail (gains) fatter than left (losses) — good |\n| = 1.0 | Symmetric |\n| < 1.0 | Left tail (losses) fatter — danger |\n\nFor a good 5-min crypto strategy, aim for tail ratio > 1.0. A ratio of 1.2-1.5 indicates favorable asymmetry.\n\n### Python Implementation\n\n```python\ndef tail_ratio(returns: np.ndarray, percentile: float = 95) -> float:\n    \"\"\"Tail ratio: right tail / left tail.\"\"\"\n    right = np.percentile(returns, percentile)\n    left = np.percentile(returns, 100 - percentile)\n\n    if left == 0:\n        return np.inf if right > 0 else 1.0\n    return abs(right / left)\n```\n\n### Sample Size\n\nRequires good estimation of the 5th and 95th percentiles. Minimum ~10K bars for rough estimate, 50K+ for stability.\n\n---\n\n## 9. Profit Factor\n\n### Formula\n\n```\nProfit Factor = sum(winning_returns) / abs(sum(losing_returns))\n```\n\nThis is computed on trade-level P&L, not bar returns.\n\n### Interpretation for 5-min Crypto\n\n| Profit Factor | Quality |\n|---|---|\n| < 1.0 | Losing strategy |\n| 1.0 - 1.2 | Marginal (may not survive costs) |\n| 1.2 - 1.5 | Acceptable |\n| 1.5 - 2.0 | Good |\n| > 2.0 | Excellent |\n\nFor 5-min crypto with typical exchange fees (~0.04-0.1% per trade), you need a profit factor of at least 1.2 to cover costs. Higher frequency = need higher profit factor.\n\n### Python Implementation\n\n```python\ndef profit_factor(trade_pnls: np.ndarray) -> float:\n    \"\"\"Profit factor from trade-level P&L.\"\"\"\n    gross_profit = np.sum(trade_pnls[trade_pnls > 0])\n    gross_loss = abs(np.sum(trade_pnls[trade_pnls < 0]))\n\n    if gross_loss == 0:\n        return np.inf if gross_profit > 0 else 0.0\n    return gross_profit / gross_loss\n```\n\n---\n\n## 10. Win Rate, Payoff Ratio, Expectancy\n\n### Formulas\n\n```\nWin Rate = # winning trades / # total trades\n\nPayoff Ratio = avg_win / abs(avg_loss)\n\nExpectancy = Win_Rate × avg_win - (1 - Win_Rate) × abs(avg_loss)\n           = avg_loss × (Win_Rate × Payoff_Ratio - (1 - Win_Rate))\n```\n\nExpectancy per dollar risked:\n\n```\nE = Win_Rate × Payoff_Ratio - (1 - Win_Rate)\n```\n\n### Minimum Win Rate Given Payoff Ratio\n\nTo be profitable (E > 0):\n\n```\nWin_Rate_min = 1 / (1 + Payoff_Ratio)\n```\n\n| Payoff Ratio | Min Win Rate |\n|---|---|\n| 0.5 | 66.7% |\n| 1.0 | 50.0% |\n| 1.5 | 40.0% |\n| 2.0 | 33.3% |\n| 3.0 | 25.0% |\n\n### After Costs\n\nAdd transaction costs to each trade and recompute. For 5-min crypto:\n- Exchange fee: ~0.04% maker, ~0.06% taker (varies)\n- Slippage: ~0.01-0.05% per side for liquid pairs\n- Round-trip cost: ~0.1-0.2%\n\nAdjusted expectancy:\n\n```\nE_adj = E - round_trip_cost_per_trade\n```\n\n### Python Implementation\n\n```python\ndef trade_statistics(trade_pnls: np.ndarray) -> dict:\n    \"\"\"Win rate, payoff ratio, expectancy.\"\"\"\n    winners = trade_pnls[trade_pnls > 0]\n    losers = trade_pnls[trade_pnls < 0]\n    flat = trade_pnls[trade_pnls == 0]\n\n    n_total = len(trade_pnls)\n    n_win = len(winners)\n    n_loss = len(losers)\n\n    win_rate = n_win / n_total if n_total > 0 else 0\n    avg_win = np.mean(winners) if n_win > 0 else 0\n    avg_loss = np.mean(losers) if n_loss > 0 else 0\n    payoff_ratio = abs(avg_win / avg_loss) if avg_loss != 0 else np.inf\n\n    expectancy = win_rate * avg_win + (1 - win_rate) * avg_loss\n    min_win_rate = 1 / (1 + payoff_ratio) if payoff_ratio != np.inf else 0\n\n    return {\n        \"win_rate\": win_rate,\n        \"avg_win\": avg_win,\n        \"avg_loss\": avg_loss,\n        \"payoff_ratio\": payoff_ratio,\n        \"expectancy\": expectancy,\n        \"min_win_rate_breakeven\": min_win_rate,\n        \"n_trades\": n_total,\n        \"n_winners\": n_win,\n        \"n_losers\": n_loss,\n        \"n_flat\": len(flat),\n    }\n```\n\n### Sample Size\n\nThe binomial standard error of win rate is:\n\n```\nSE(WR) = sqrt(WR × (1 - WR) / N)\n```\n\nFor WR=55% with 2% margin of error at 95% confidence: N = (1.96/0.02)^2 × 0.55 × 0.45 ≈ 2,376 trades. In practice, aim for 1,000+ trades minimum.\n\n---\n\n## 11. Ulcer Index\n\n### Formula\n\nThe Ulcer Index measures the depth and duration of drawdowns:\n\n```\nR_i = 100 × (Price_i / max(Price_0..i) - 1)\n\nUlcer Index = sqrt(mean(R_i^2))\n```\n\nThis is the RMS (root mean square) of percentage drawdowns. Unlike max drawdown which captures one point, the Ulcer Index penalizes both the magnitude and persistence of drawdowns.\n\n### Martin Ratio (Ulcer Performance Index)\n\n```\nMartin Ratio = (Return - Rf) / Ulcer_Index\n```\n\nThis is Sharpe but using Ulcer Index instead of standard deviation. It only penalizes downside movements.\n\n### Interpretation\n\n| Ulcer Index | Meaning |\n|---|---|\n| < 2% | Very low stress |\n| 2-5% | Moderate |\n| 5-10% | Uncomfortable |\n| > 10% | High stress |\n\n### Python Implementation\n\n```python\ndef ulcer_index(equity_curve: np.ndarray) -> float:\n    \"\"\"Ulcer Index: RMS of percentage drawdowns.\"\"\"\n    running_max = np.maximum.accumulate(equity_curve)\n    pct_drawdown = 100 * (equity_curve / running_max - 1)\n    return np.sqrt(np.mean(pct_drawdown**2))\n\n\ndef martin_ratio(\n    returns: np.ndarray,\n    rf_annual: float = 0.0,\n) -> float:\n    \"\"\"Martin Ratio (UPI): excess return / Ulcer Index.\"\"\"\n    equity = np.cumprod(1 + returns)\n    ann_return = equity[-1] ** (BARS_PER_YEAR / len(returns)) - 1\n    ui = ulcer_index(equity)\n\n    if ui == 0:\n        return np.inf if ann_return > rf_annual else 0.0\n    return (ann_return - rf_annual) / (ui / 100)\n```\n\n---\n\n## 12. Information Ratio\n\n### Formula\n\n```\nIR = (R_p - R_b) / TE\n\nTE = std(R_p - R_b)   # tracking error\n```\n\nWhere R_b is the benchmark return.\n\n### Benchmark for Crypto Prediction Strategy\n\nOptions, ranked by appropriateness:\n\n1. **Buy-and-hold BTC** — if your strategy trades BTC, this is the natural benchmark. Your alpha is the return above simply holding.\n2. **Buy-and-hold the traded asset** — for any specific pair.\n3. **Equal-weight crypto index** — if trading multiple assets.\n4. **Risk-free rate** — if you consider the strategy market-neutral (in which case IR degenerates to Sharpe).\n5. **Zero** — for an absolute return strategy. IR then equals Sharpe with Rf=0.\n\nFor a prediction strategy that goes long/short or varies position size, buy-and-hold of the underlying is the best benchmark.\n\n### Interpretation\n\n| IR | Quality |\n|---|---|\n| < 0.0 | Worse than benchmark |\n| 0.0 - 0.5 | Weak alpha |\n| 0.5 - 1.0 | Good alpha |\n| > 1.0 | Excellent |\n\n### Annualization\n\n```\nIR_annual = IR_per_bar × sqrt(BARS_PER_YEAR)\n```\n\n### Python Implementation\n\n```python\ndef information_ratio(\n    returns: np.ndarray,\n    benchmark_returns: np.ndarray,\n    annualize: bool = True,\n) -> float:\n    \"\"\"Information ratio vs benchmark.\"\"\"\n    active = returns - benchmark_returns\n    te = np.std(active, ddof=1)\n\n    if te == 0:\n        return np.inf if np.mean(active) > 0 else 0.0\n\n    ir = np.mean(active) / te\n    if annualize:\n        ir *= SQRT_BARS\n    return ir\n```\n\n---\n\n## 13. Rachev Ratio\n\n### Formula\n\n```\nRachev(alpha, beta) = CVaR_alpha(+R) / CVaR_beta(-R)\n```\n\nWhere CVaR_alpha(+R) is the expected gain in the top alpha% of returns, and CVaR_beta(-R) is the expected loss in the bottom beta% of returns.\n\nTypically alpha = beta = 0.05.\n\n### Why It Matters\n\n- Directly compares tail risk vs tail reward\n- If > 1, the strategy's best outcomes are larger (in magnitude) than its worst outcomes\n- Robust to fat tails (unlike Sharpe)\n\n### Interpretation\n\n| Rachev | Meaning |\n|---|---|\n| < 1.0 | Tail losses exceed tail gains |\n| = 1.0 | Symmetric tails |\n| > 1.0 | Tail gains exceed tail losses — favorable |\n| > 1.5 | Strong positive tail asymmetry |\n\n### Python Implementation\n\n```python\ndef rachev_ratio(\n    returns: np.ndarray,\n    alpha: float = 0.05,\n    beta: float = 0.05,\n) -> float:\n    \"\"\"Rachev ratio: CVaR of gains / CVaR of losses.\"\"\"\n    # Top alpha% gains\n    gain_threshold = np.percentile(returns, (1 - alpha) * 100)\n    top_gains = returns[returns >= gain_threshold]\n    avg_top_gain = np.mean(top_gains) if len(top_gains) > 0 else 0\n\n    # Bottom beta% losses\n    loss_threshold = np.percentile(returns, beta * 100)\n    bottom_losses = returns[returns <= loss_threshold]\n    avg_bottom_loss = np.mean(bottom_losses) if len(bottom_losses) > 0 else 0\n\n    if avg_bottom_loss == 0:\n        return np.inf if avg_top_gain > 0 else 1.0\n    return abs(avg_top_gain / avg_bottom_loss)\n```\n\n---\n\n## 14. Kappa Ratio\n\n### Formula\n\nThe Kappa ratio of order n generalizes Sharpe (n=2) and Sortino (n=2 with downside):\n\n```\nKappa_n = (mean(R) - MAR) / LPM_n^(1/n)\n\nLPM_n = mean(max(MAR - R, 0)^n)\n```\n\nWhere LPM_n is the n-th Lower Partial Moment.\n\n- **Kappa 1** = Omega - 1 (equivalent to Omega ratio)\n- **Kappa 2** = Sortino ratio\n- **Kappa 3** penalizes large downside deviations even more heavily (cubed)\n\n### Why Kappa 3 Is Useful for Crypto\n\nHigher-order Kappa puts more weight on extreme downside events. For fat-tailed crypto returns, Kappa 3 differentiates strategies that have similar Sortino ratios but different exposure to tail risk.\n\n### Python Implementation\n\n```python\ndef kappa_ratio(\n    returns: np.ndarray,\n    n: int = 3,\n    mar_annual: float = 0.0,\n    annualize: bool = True,\n) -> float:\n    \"\"\"Kappa ratio of order n. n=1: Omega-1, n=2: Sortino, n=3: Kappa3.\"\"\"\n    mar_bar = (1 + mar_annual) ** (1 / BARS_PER_YEAR) - 1\n    excess = returns - mar_bar\n\n    downside = np.maximum(mar_bar - returns, 0)\n    lpm_n = np.mean(downside**n)\n\n    if lpm_n == 0:\n        return np.inf if np.mean(excess) > 0 else 0.0\n\n    kappa = np.mean(excess) / (lpm_n ** (1 / n))\n    if annualize:\n        kappa *= SQRT_BARS\n    return kappa\n```\n\n---\n\n## 15.
