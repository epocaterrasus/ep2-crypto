Market Impact Models\n\n### Square Root Law (Empirical Standard)\n\nThe most widely used market impact model in crypto:\n\n```\nImpact (bps) = eta * sigma * sqrt(Q / V)\n```\n\nWhere:\n- `eta` (eta): impact coefficient, typically **0.1 to 0.5** for crypto (higher than equities due to less depth)\n- `sigma`: volatility (e.g., 5-min realized vol in bps)\n- `Q`: order quantity\n- `V`: average daily volume\n\n### Almgren-Chriss Model\n\nThe Almgren-Chriss framework splits impact into:\n1. **Temporary impact**: transient price displacement that reverts (from consuming liquidity)\n2. **Permanent impact**: lasting price change from information content of the trade\n\n```python\nimport math\nimport logging\nfrom dataclasses import dataclass\n\nlogger = logging.getLogger(__name__)\n\n\n@dataclass\nclass MarketImpactEstimate:\n    temporary_impact_bps: float\n    permanent_impact_bps: float\n    total_impact_bps: float\n    optimal_execution_time_sec: float\n    estimated_cost_usd: float\n\n\nclass MarketImpactModel:\n    \"\"\"\n    Market impact estimation using Square Root Law and Almgren-Chriss.\n\n    Calibrated for BTCUSDT perpetual futures.\n    \"\"\"\n\n    def __init__(\n        self,\n        eta: float = 0.25,          # Square root law: temporary impact\n        temporary_bps = self.eta * volatility_5min_bps * math.pow(\n            participation, self.alpha\n        )\n\n        # Permanent impact (linear in participation for small orders)\n        permanent_bps = self.gamma * volatility_5min_bps * participation\n\n        total_bps = temporary_bps + permanent_bps\n        cost_usd = total_bps / 10000 * order_size_btc * current_price\n\n        # Almgren-Chriss optimal execution time\n        # T* = sqrt(risk_aversion * sigma^2 * Q / (2 * eta))\n        sigma_per_sec = (volatility_5min_bps / 10000) / math.sqrt(300)  # per-second vol\n        if self.eta > 0 and sigma_per_sec > 0:\n            optimal_time = math.sqrt(\n                self.risk_aversion * (sigma_per_sec ** 2) * order_size_btc\n                / (2 * self.eta / 10000)\n            )\n            # Clamp to reasonable range\n            optimal_time = max(1.0, min(optimal_time, 300.0))\n        else:\n            optimal_time = 60.0\n\n        return MarketImpactEstimate(\n            temporary_impact_bps=round(temporary_bps, 3),\n            permanent_impact_bps=round(permanent_bps, 3),\n            total_impact_bps=round(total_bps, 3),\n            optimal_execution_time_sec=round(optimal_time, 1),\n            estimated_cost_usd=round(cost_usd, 2),\n        )\n\n    def calibrate_from_fills(self, fills: list[dict]) -> dict:\n        \"\"\"\n        Calibrate eta and gamma from historical execution data.\n\n        Each fill dict should have:\n            - order_size_btc\n            - daily_volume_btc\n            - volatility_5min_bps\n            - observed_impact_bps (arrival slippage)\n        \"\"\"\n        if len(fills) < 10:\n            logger.warning(\"Need at least 10 fills to calibrate\")\n            return {\"eta\": self.eta, \"gamma\": self.gamma}\n\n        # Reasonable bounds\n\n        logger.info(\n            \"Impact model calibrated\",\n            extra={\"eta\": self.eta, \"num_fills\": len(fills)},\n        )\n\n        return {\"eta\": self.eta, \"gamma\": self.gamma}\n\n\n# Quantitative estimates for BTCUSDT perps\ndef print_impact_table():\n    \"\"\"Show expected impact for various order sizes.\"\"\"\n    model = MarketImpactModel(eta=0.25)\n\n    # Order Book-Aware Execution\n\n### Concept\n\nInstead of blindly placing orders, inspect the order book to:\n- Place limit orders at price levels with existing liquidity (hide behind whales)\n- Avoid sweeping through thin levels that cause excessive slippage\n- Detect when the book is one-sided (imbalanced) and adjust aggression\n\n```python\nimport logging\nfrom dataclasses import dataclass\n\nlogger = logging.getLogger(__name__)\n\n\n@dataclass\nclass BookLevel:\n    price: float\n    quantity: float\n    cumulative_qty: float\n\n\n@dataclass\nclass BookAnalysis:\n    bid_depth_10bps: float     # Calculate depth within 10 bps\n        threshold_10bps = mid * 0.001\n        bid_depth = sum(\n            qty for price, qty in bids if price >= mid - threshold_10bps\n        )\n        ask_depth = sum(\n            qty for price, qty in asks if price <= mid + threshold_10bps\n        )\n\n        total_depth = bid_depth + ask_depth\n        imbalance = (bid_depth - ask_depth) / total_depth if total_depth > 0 else 0\n\n        # Not enough liquidity in visible book\n            logger.warning(\n                \"Insufficient book depth\",\n                extra={\"remaining\": remaining, \"order_size\": order_size},\n            )\n            return 999.0  # Flag as dangerous\n\n        vwap = total_cost / order_size\n        direction = 1 if side == \"buy\" else -1\n        return direction * (vwap - mid) / mid * 10000\n\n    def _find_optimal_placement(\n        self, our_side_levels: list, order_size: float, side: str, imbalance: float\n    ) -> float:\n        \"\"\"\n        Find the best price to place our limit order.\n        Strategy: place behind large resting orders for cover.\n        \"\"\"\n        if not our_side_levels:\n            return 0.0\n\n        best_price = our_side_levels[0][0]\n\n        # Look for \"walls\" - large resting orders in the first 5 levels\n        avg_size = sum(qty for _, qty in our_side_levels[:10]) / min(10, len(our_side_levels))\n        wall_threshold = avg_size * 3  # 3x average size = wall\n\n        for price, qty in our_side_levels[:10]:\n            if qty >= wall_threshold:\n                # Place just in front of the wall (between wall and mid)\n                if side == \"buy\":\n                    # Wall is a bid wall, place just above it\n                    return price + 0.1  # One tick in front\n                else:\n                    return price - 0.1\n                break\n\n        # No wall found: if imbalance favors us, be patient (post deeper)\n        # If imbalance is against us, be aggressive (post at top)\n        if side == \"buy\":\n            if imbalance > 0.3:\n                # Position Management\n\n### Scaling In/Out and Pyramiding\n\n```python\nimport logging\nfrom dataclasses import dataclass, field\nfrom enum import Enum\n\nlogger = logging.getLogger(__name__)\n\n\nclass SignalStrength(Enum):\n    VERY_STRONG = 4\n    STRONG = 3\n    MODERATE = 2\n    WEAK = 1\n\n\n@dataclass\nclass PositionState:\n    side: str = \"\"             # \"long\" or \"short\" or \"\"\n    total_quantity: float = 0.0\n    avg_entry_price: float = 0.0\n    entries: list[dict] = field(default_factory=list)\n    unrealized_pnl: float = 0.0\n    max_quantity: float = 0.0  # Peak position size (for drawdown calc)\n\n\nclass PositionManager:\n    \"\"\"\n    Manages position scaling based on signal strength and P&L.\n\n    Pyramiding rules:\n    1. Hedging Strategies\n\n### When Hedging Makes Sense for a 5-Min Prediction System\n\n| Scenario | Hedge? | Method |\n|---|---|---|\n| High-confidence directional signal | No | Take directional exposure |\n| Uncertain signal with vol opportunity | Yes | Long straddle via perps + spot |\n| Funding rate arbitrage | Yes | Long spot + short perp (or vice versa) |\n| Cross-exchange price dislocation | Yes | Long on cheap exchange, short on expensive |\n| Overnight/weekend risk reduction | Yes | Reduce net exposure via hedge |\n| Correlated pair divergence | Yes | Long BTCUSDT, short ETHUSDT (pair trade) |\n\n### Funding Rate Arbitrage\n\nBinance perps charge funding every 8 hours. When funding is high positive (longs pay shorts), you can:\n- Go long spot BTC\n- Go short BTCUSDT perps\n- Collect funding rate with delta-neutral exposure\n\nTypical funding rate: 0.01% per 8 hours = **0.03% per day = ~11% annualized** in calm markets. ~0.28% total\n\n        # Minimum holding period to be profitable\n        if rate > 0:\n            periods_to_breakeven = round_trip_cost / rate\n            hours_to_breakeven = periods_to_breakeven * 8\n        else:\n            hours_to_breakeven = float(\"inf\")\n\n        return {\n            \"current_funding_rate\": rate,\n            \"annualized_return\": round(annualized * 100, 2),\n            \"round_trip_cost_pct\": round(round_trip_cost * 100, 3),\n            \"hours_to_breakeven\": round(hours_to_breakeven, 1),\n            \"is_profitable\": rate >= self.min_funding_rate,\n        }\n\n    async def enter_hedge(self, quantity_btc: float) -> dict:\n        \"\"\"Enter funding arb: long spot, short perp.\"\"\"\n        opportunity = await self.check_opportunity()\n        if not opportunity[\"is_profitable\"]:\n            logger.info(\"Funding rate too low\", extra=opportunity)\n            return {\"status\": \"skipped\", \"reason\": \"funding_too_low\"}\n\n        #
