Position Sizing Methods\n\n### 1. Fixed Fractional\n\n```\nPosition Size = Account Balance * Risk Fraction / (Entry - StopLoss)\n```\n\n**Typical parameters:**\n- Risk fraction: 1–2% of account per trade\n- Simple, robust, prevents ruin\n\n**Pros:** Easy to implement, limits downside.\n**Cons:** Doesn't adapt to changing edge or volatility.\n\n### 2. Kelly Criterion\n\n**Exact formula (for binary outcomes):**\n```\nf* = (p * b - q) / b\n```\nWhere:\n- `f*` = optimal fraction of capital to bet\n- `p` = probability of winning\n- `b` = win/loss ratio (average win / average loss)\n- `q` = 1 - p\n\n**For continuous returns:**\n```\nf* = mu / sigma^2\n```\nWhere `mu` = expected return per trade, `sigma^2` = variance of returns.\n\n**Practical adjustments:**\n- **Half-Kelly (`f*/2`):** Standard practice. Achieves 75% of the growth rate of full Kelly with massively reduced drawdowns and variance.\n- **Quarter-Kelly (`f*/4`):** Even more conservative. Recommended for crypto due to fat tails.\n\n**Example:**\n- Win rate: 55%, avg win: 20 bps, avg loss: 15 bps\n- `b = 20/15 = 1.333`\n- `f* = (0.55 * 1.333 - 0.45) / 1.333 = (0.733 - 0.45) / 1.333 = 0.212`\n- Full Kelly: 21.2% of capital per trade\n- Half Kelly: 10.6%\n- Quarter Kelly: 5.3%\n\n**Warning:** Kelly assumes you know the true parameters. Estimation error in `p` and `b` makes full Kelly extremely dangerous. In practice, your estimated edge is almost always overstated.\n\n### 3. Volatility-Adjusted (ATR-Based)\n\n```\nPosition Size = (Account * Risk%) / (ATR_n * Multiplier)\n```\n\nWhere:\n- `ATR_n` = Average True Range over n periods\n- `Multiplier` = 1.5–3.0 (determines stop distance in ATR units)\n\n**Example:**\n- Account: $100K, Risk: 1%, ATR(14) on 5-min: $50, Multiplier: 2\n- Position Size = ($100K * 0.01) / ($50 * 2) = $1000 / $100 = 10 contracts/units\n- In volatile markets (ATR = $100): 5 contracts\n- In calm markets (ATR = $25): 20 contracts\n\n### Which Works Best for 5-min Crypto?\n\n**Volatility-adjusted sizing with a Kelly-informed cap** is the consensus best practice:\n\n1. Use ATR to set the base position size (adapts to current volatility).\n2. Use model confidence (prediction probability) to scale within that base.\n3. Cap maximum position at half-Kelly.\n4. Never risk more than 1–2% of account per trade.\n\n```python\ndef compute_position_size(account_balance, model_confidence, atr, \n                          avg_win, avg_loss, win_rate):\n    # Volatility-based sizing\n    risk_per_trade = account_balance * 0.01  # 1% risk\n    vol_size = risk_per_trade / (atr * 2)  # 2 ATR stop\n    \n    # Kelly-based cap\n    b = avg_win / avg_loss\n    kelly_f = (win_rate * b - (1 - win_rate)) / b\n    half_kelly_size = account_balance * (kelly_f / 2)\n    \n    # Confidence scaling\n    confidence_scalar = (model_confidence - 0.5) * 2  # 0 to 1 for 50-100%\n    \n    # Final size: minimum of vol-based and Kelly cap, scaled by confidence\n    position_size = min(vol_size, half_kelly_size) * confidence_scalar\n    \n    # Hard cap at 5% of account\n    position_size = min(position_size, account_balance * 0.05)\n    \n    return max(position_size, 0)\n```\n\n---\n\n## 10. Risk Management: Stops and Exits\n\n### Do Hard Stop-Losses Help or Hurt ML-Based 5-min Systems?\n\n**The evidence is mixed but leans toward: tight stops hurt, wide stops help.**\n\n**Arguments against tight stops (< 1 ATR):**\n- 5-min candles are noisy. Catastrophic stop (mandatory):** Set a wide stop at 3–5 ATR or 1–2% of account. This is not for regular exits; it's for disaster protection.\n\n**2. Time-based exit (strongly recommended for 5-min):** Exit after N bars if the predicted move hasn't materialized. For 5-min predictions:\n- If predicting next-bar direction: exit after 1–2 bars.\n- If predicting 30-min horizon: exit after 6–10 bars.\n- Time stops avoid holding losing positions that bleed via funding/opportunity cost.\n\n**3.
