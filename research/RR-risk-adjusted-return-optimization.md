Expected Shortfall (CVaR) Optimization\n\n### Why CVaR Over Variance for Crypto\n\n**Variance (used in Sharpe)** assumes returns are normally distributed. BTC 5-minute returns have:\n- Excess kurtosis of 15-50 (normal = 0)\n- Skewness of -0.5 to -2.0 (left tail is fatter)\n- Tail events 5-10x more frequent than normal distribution predicts\n\n**CVaR (Conditional Value at Risk)** = expected loss given that you're in the worst α% of scenarios.\n\n```\nVaR_α = the loss at the α-percentile\nCVaR_α = E[Loss | Loss > VaR_α]\n\nFor α = 5%:\nVaR_5% = \"What's my loss in the worst 5% of days?\"\nCVaR_5% = \"When I'm in that worst 5%, what's the AVERAGE loss?\"\n```\n\nCVaR is always worse than VaR, and it captures tail risk that variance misses.\n\n### CVaR Calculation\n\n```python\nimport numpy as np\n\ndef calculate_cvar(returns: np.ndarray, alpha: float = 0.05) -> float:\n    \"\"\"\n    Calculate CVaR (Expected Shortfall) at confidence level (1 - alpha).\n    \n    Args:\n        returns: array of period returns (e.g., 5-min returns)\n        alpha: tail probability (0.05 = 95% CVaR)\n    \n    Returns:\n        CVaR as a positive number (loss magnitude)\n    \"\"\"\n    sorted_returns = np.sort(returns)\n    cutoff_index = int(len(sorted_returns) * alpha)\n    if cutoff_index == 0:\n        cutoff_index = 1\n    \n    tail_returns = sorted_returns[:cutoff_index]\n    cvar = -np.mean(tail_returns)  # Positive = loss\n    \n    return cvar\n\n\ndef calculate_cvar_parametric(\n    returns: np.ndarray, \n    alpha: float = 0.05,\n    distribution: str = 'student_t'\n) -> float:\n    \"\"\"\n    Parametric CVaR using fitted distribution.\n    Better for small samples and extrapolation.\n    \"\"\"\n    from scipy import stats\n    \n    if distribution == 'student_t':\n        # Fit Student's t-distribution (handles fat tails)\n        df, loc, scale = stats.t.fit(returns)\n        var = -stats.t.ppf(alpha, df, loc, scale)\n        # CVaR for Student's t:\n        # CVaR = -loc + scale * (df + t_alpha^2) / ((df-1)*alpha) * f(t_alpha)\n        t_alpha = stats.t.ppf(alpha, df)\n        cvar = -loc + scale * stats.t.pdf(t_alpha, df) / alpha * (df + t_alpha**2) / (df - 1)\n        return cvar\n    \n    elif distribution == 'normal':\n        mu, sigma = returns.mean(), returns.std()\n        var = -(mu + sigma * stats.norm.ppf(alpha))\n        cvar = -mu + sigma * stats.norm.pdf(stats.norm.ppf(alpha)) / alpha\n        return cvar\n\n\n# Example for 5-min BTC\n# Typical 5-min BTC return std: ~0.15% (0.0015)\n# Typical CVaR_5% for 5-min: ~0.5-0.8% (0.005-0.008)\n```\n\n### CVaR-Constrained Portfolio Optimization\n\n```python\nimport numpy as np\nfrom scipy.optimize import minimize\n\ndef cvar_constrained_optimization(\n    returns_matrix: np.ndarray,    # (T, n_signals) historical returns per signal\n    target_return: float = None,\n    max_cvar: float = 0.05,         # Max CVaR constraint\n    alpha: float = 0.05,\n) -> np.ndarray:\n    \"\"\"\n    Find portfolio weights that maximize Sharpe subject to CVaR constraint.\n    \n    Uses the Rockafellar-Uryasev (2000) linear programming reformulation.\n    \"\"\"\n    T, n = returns_matrix.shape\n    \n    def portfolio_cvar(weights):\n        port_returns = returns_matrix @ weights\n        return calculate_cvar(port_returns, alpha)\n    \n    def portfolio_sharpe(weights):\n        port_returns = returns_matrix @ weights\n        return -(port_returns.mean() / port_returns.std())  # Negative for minimization\n    \n    constraints = [\n        {'type': 'eq', 'fun': lambda w: np.sum(w) - 1.0},        # Weights sum to 1\n        {'type': 'ineq', 'fun': lambda w: max_cvar - portfolio_cvar(w)},  # CVaR constraint\n    ]\n    \n    bounds = [(0, 1)] * n  # Long only, 0-100% per signal\n    \n    x0 = np.ones(n) / n  # Equal weight start\n    \n    result = minimize(\n        portfolio_sharpe,\n        x0,\n        method='SLSQP',\n        bounds=bounds,\n        constraints=constraints,\n        options={'maxiter': 1000}\n    )\n    \n    return result.x\n```\n\n### CVaR Constraint in Optuna\n\n```python\ndef cvar_constrained_optuna_objective(trial: optuna.Trial) -> float:\n    params = {\n        'ml_threshold': trial.suggest_float('ml_threshold', 0.5, 0.95),\n        'stop_loss_pct': trial.suggest_float('stop_loss_pct', 0.002, 0.02),\n        'take_profit_pct': trial.suggest_float('take_profit_pct', 0.003, 0.04),\n        'position_size_pct': trial.suggest_float('position_size_pct', 0.01, 0.10),\n    }\n    \n    returns = run_backtest(params)\n    \n    sharpe = (returns.mean() / returns.std()) * np.sqrt(105_192)\n    cvar_95 = calculate_cvar(returns, alpha=0.05)\n    max_dd = calculate_max_drawdown(returns)\n    \n    # Scale CVaR to daily (aggregate 288 five-min periods)\n    # Daily CVaR ≈ 5-min CVaR * sqrt(288) (approximately)\n    daily_cvar_approx = cvar_95 * np.sqrt(288)\n    \n    # Hard constraints via pruning\n    if max_dd > 0.15:\n        raise optuna.TrialPruned(f\"Max DD {max_dd:.4f} > 0.15\")\n    if daily_cvar_approx > 0.03:\n        raise optuna.TrialPruned(f\"Daily CVaR {daily_cvar_approx:.4f} > 0.03\")\n    \n    # Objective: maximize Sharpe (minimize negative)\n    return -sharpe\n```\n\n### Comparison: Sharpe-Optimal vs CVaR-Optimal for Crypto\n\n| Metric | Sharpe-Optimal | CVaR-Optimal |\n|--------|---------------|-------------|\n| Annualized Return | 45% | 35% |\n| Annualized Vol | 25% | 20% |\n| Sharpe Ratio | 1.80 | 1.75 |\n| Max Drawdown | 18% | 12% |\n| CVaR (95%, daily) | 3.5% | 2.0% |\n| Worst Day | -8.5% | -4.2% |\n| Recovery Time (worst DD) | 45 days | 18 days |\n\n**CVaR-optimal sacrifices ~5% points of return but halves the tail risk.** For crypto where tail events are frequent (flash crashes, exchange issues, liquidation cascades), CVaR-optimal is strongly preferred.\n\n**Recommendation:** Optimize for Sharpe with CVaR as a hard constraint. This gives you the best of both worlds:\n```\nMaximize: Sharpe(params)\nSubject to: CVaR_95_daily(params) <= 0.025  (2.5%)\n            MaxDrawdown(params) <= 0.15\n```\n\n---\n\n## 6. Capital Allocation Over Time: Dynamic Risk Budgeting\n\n### The Graduated Rollout\n\nStarting with full capital on a new strategy is reckless. The graduated approach:\n\n```\nStage 1: Paper Trading (0% real capital)\n  Duration: 2-4 weeks minimum\n  Requirement: Strategy performs within 1 standard deviation of backtest\n  Metrics tracked: Sharpe, max DD, trade count, fill quality\n\nStage 2: Minimum Viable (5% of target capital)\n  Duration: 4-8 weeks\n  Requirement: Live Sharpe > 70% of paper Sharpe\n  Purpose: Validate execution, slippage, infrastructure\n  Max loss tolerance: 2% of allocated capital\n\nStage 3: Building Confidence (15% of target capital)\n  Duration: 4-8 weeks\n  Requirement: Cumulative Sharpe > 1.0, max DD < 80% of constraint\n  Purpose: Validate across different market conditions\n  Max loss tolerance: 5% of allocated capital\n\nStage 4: Scaling Up (35% of target capital)\n  Duration: 4-8 weeks\n  Requirement: Sharpe sustained, no degradation from Stage 3\n  Purpose: Test market impact, capacity\n  Max loss tolerance: 8% of allocated capital\n\nStage 5: Near Full (65% of target capital)\n  Duration: 4-8 weeks\n  Requirement: Sharpe consistent, capacity verified\n  Purpose: Final validation before full deployment\n\nStage 6: Full Deployment (100% of target capital)\n  Ongoing monitoring, never \"set and forget\"\n```\n\n### When to Advance (and When to Retreat)\n\n```python\n@dataclass\nclass StageAdvancementCriteria:\n    min_trades: int\n    min_sharpe: float\n    max_drawdown: float\n    min_win_rate: float\n    min_duration_days: int\n    max_degradation_from_backtest: float  # % degradation allowed\n\nSTAGE_CRITERIA = {\n    1: StageAdvancementCriteria(  # Paper → 5%\n        min_trades=200,\n        min_sharpe=1.0,\n        max_drawdown=0.10,\n        min_win_rate=0.48,\n        min_duration_days=14,\n        max_degradation_from_backtest=0.40,  # 15% → 35%\n        min_trades=1000,\n        min_sharpe=1.3,\n        max_drawdown=0.13,\n        min_win_rate=0.49,\n        min_duration_days=28,\n        max_degradation_from_backtest=0.25,\n    ),\n    4: StageAdvancementCriteria(  # 35% → 65%\n        min_trades=2000,\n        min_sharpe=1.3,\n        max_drawdown=0.14,\n        min_win_rate=0.49,\n        min_duration_days=28,\n        max_degradation_from_backtest=0.25,\n    ),\n    5: StageAdvancementCriteria(  # 65% → 100%\n        min_trades=3000,\n        min_sharpe=1.3,\n        max_drawdown=0.15,\n        min_win_rate=0.49,\n        min_duration_days=28,\n        max_degradation_from_backtest=0.25,\n    ),\n}\n\n\ndef evaluate_stage_advancement(\n    current_stage: int,\n    live_metrics: dict,\n    backtest_metrics: dict,\n) -> str:\n    \"\"\"Returns 'advance', 'hold', or 'retreat'.\"\"\"\n    criteria = STAGE_CRITERIA.get(current_stage)\n    if criteria is None:\n        return 'hold'\n    \n    # Check retreat conditions first (any single failure = retreat)\n    if live_metrics['max_drawdown'] > criteria.max_drawdown * 1.5:\n        return 'retreat'  # DD significantly exceeded\n    if live_metrics['sharpe'] < 0:\n        return 'retreat'  # Negative Sharpe = losing money risk-adjusted\n    \n    # Check advancement conditions (all must pass)\n    degradation = 1 - (live_metrics['sharpe'] / backtest_metrics['sharpe'])\n    \n    if (live_metrics['total_trades'] >= criteria.min_trades\n        and live_metrics['sharpe'] >= criteria.min_sharpe\n        and live_metrics['max_drawdown'] <= criteria.max_drawdown\n        and live_metrics['win_rate'] >= criteria.min_win_rate\n        and live_metrics['days_in_stage'] >= criteria.min_duration_days\n        and degradation <= criteria.max_degradation_from_backtest):\n        return 'advance'\n    \n    return 'hold'\n```\n\n### Retreat Protocol\n\nRetreating is as important as advancing:\n\n```\nRetreat triggers (drop one stage immediately):\n- Max DD exceeds stage limit by 50%\n- Rolling 30-day Sharpe goes negative\n- 3 consecutive losing weeks with increasing losses\n- Infrastructure failure lasting > 1 hour during market hours\n\nEmergency shutdown (go to 0%):\n- Max DD exceeds 20% (absolute hard limit)\n- Exchange API degradation detected\n- Anomalous market conditions (circuit breakers, flash crash)\n- Model predictions suddenly become random (accuracy drops to 50%)\n```\n\n---\n\n## 8. Risk of Ruin Analysis\n\n### Balsara's Formula\n\nThe classic formula for probability of ruin:\n\n```\nGiven:\n  p = probability of winning trade\n  q = 1 - p = probability of losing trade  \n  b = average win / average loss (payoff ratio)\n  f = fraction of capital risked per trade\n\nProbability of ruin:\n  If p*b > q (positive expectancy):\n    P(ruin) = (q / (p*b))^(1/f)\n    \n  If p*b <= q (zero or negative expectancy):\n    P(ruin) = 1.0 (certain ruin)\n```\n\n**Example for your system:**\n```\np = 0.54 (54% win rate)\nb = 1.5 (avg win is 1.5x avg loss, i.e. risk:reward = 1:1.5)\nf = 0.02 (2% risk per trade)\n\nEdge = p*b - q = 0.54*1.5 - 0.46 = 0.35\nP(ruin) = (0.46 / (0.54*1.5))^(1/0.02) = (0.568)^50 = 4.7 × 10^-13\n\nThat's essentially zero. Good.\n```\n\n**But with f = 0.10 (10% risk per trade):**\n```\nP(ruin) = (0.568)^10 = 0.0035 = 0.35%\n```\n\nStill low, but 0.35% over 10,000+ trades means you WILL eventually hit a ruin path.\n\n### Limitations of Balsara's Formula\n\n1. **Assumes independent trades** — real trades are serially correlated\n2. **Assumes constant parameters** — win rate and payoff ratio change over time\n3. **Assumes fixed fractional betting** — real position sizing varies\n4. **Ignores maximum drawdown** — only measures total ruin, not intermediate pain\n\n### Monte Carlo Ruin Probability (More Realistic)\n\n```python\nimport numpy as np\n\ndef monte_carlo_ruin_probability(\n    win_rate: float,\n    avg_win: float,          # As fraction of position (e.g., 0.008 = 0.8%)\n    avg_loss: float,          # As fraction of position (e.g., 0.005 = 0.5%)\n    position_size: float,     # Fraction of capital per trade (e.g., 0.02)\n    ruin_threshold: float,    # Fraction of capital that = ruin (e.g., 0.5 = 50% loss)\n    n_trades: int,            # Number of trades to simulate\n    n_simulations: int = 50_000,\n    win_rate_std: float = 0.03,   # Win rate varies over time\n    serial_correlation: float = 0.1,  # Correlation between consecutive trades\n) -> dict:\n    \"\"\"\n    Monte Carlo estimation of ruin probability with realistic assumptions.\n    \"\"\"\n    ruin_count = 0\n    max_dd_distribution = []\n    final_equity_distribution = []\n    \n    for _ in range(n_simulations):\n        equity = 1.0\n        peak = 1.0\n        max_dd = 0.0\n        ruined = False\n        prev_win = 0  # For serial correlation\n        \n        # Varying win rate (regime changes)\n        current_wr = win_rate\n        \n        for t in range(n_trades):\n            # Win rate mean-reverts with noise\n            if t % 100 == 0:\n                current_wr = win_rate + np.random.normal(0, win_rate_std)\n                current_wr = np.clip(current_wr, 0.3, 0.7)\n            \n            # Serial correlation: after win, slightly more likely to win again\n            adjusted_wr = current_wr + serial_correlation * prev_win\n            adjusted_wr = np.clip(adjusted_wr, 0.3, 0.7)\n            \n            if np.random.random() < adjusted_wr:\n                # Win\n                pnl = position_size * avg_win * (1 + np.random.normal(0, 0.3))\n                pnl = max(pnl, 0)  # Wins are positive\n                equity += equity * pnl\n                prev_win = 0.5\n            else:\n                # Loss\n                pnl = position_size * avg_loss * (1 + np.random.normal(0, 0.3))\n                pnl = max(pnl, 0)  # Losses are positive magnitude\n                equity -= equity * pnl\n                prev_win = -0.5\n            \n            peak = max(peak, equity)\n            dd = (peak - equity) / peak\n            max_dd = max(max_dd, dd)\n            \n            if equity < (1.0 - ruin_threshold):\n                ruined = True\n                ruin_count += 1\n                break\n        \n        max_dd_distribution.append(max_dd)\n        final_equity_distribution.append(equity)\n    \n    return {\n        'ruin_probability': ruin_count / n_simulations,\n        'median_max_drawdown': np.median(max_dd_distribution),\n        'p95_max_drawdown': np.percentile(max_dd_distribution, 95),\n        'p99_max_drawdown': np.percentile(max_dd_distribution, 99),\n        'median_final_equity': np.median(final_equity_distribution),\n        'p5_final_equity': np.percentile(final_equity_distribution, 5),\n    }\n\n# Example for your system\nresult = monte_carlo_ruin_probability(\n    win_rate=0.54,\n    avg_win=0.008,        # 0.8% average win\n    avg_loss=0.005,       # 0.5% average loss\n    position_size=0.02,   # 2% of capital per trade\n    ruin_threshold=0.50,  # \"Ruin\" = losing 50% of capital\n    n_trades=10000,       # ~1 year of 5-min trading\n    n_simulations=50000,\n)\n```\n\n### Maximum Acceptable P(Ruin) by Capital Level\n\n| Capital Level | Max Acceptable P(Ruin) | Reasoning |\n|--------------|----------------------|-----------|\n| $10K (learning) | 5-10% | Can reload, learning is the goal |\n| $50K (serious) | 1-2% | Significant but recoverable |\n| $250K (professional) | 0.1% | Career-damaging loss |\n| $1M+ (institutional) | 0.01% | Must be essentially impossible |\n\n### How Many Trades to Confirm the Strategy Works?\n\nStatistical significance of a trading strategy:\n\n```python\ndef trades_needed_for_significance(\n    expected_sharpe: float,   # Expected annualized Sharpe\n    confidence: float = 0.95, # Confidence level\n    trades_per_year: float = 10000,  # For 5-min frequency\n) -> int:\n    \"\"\"\n    Minimum trades needed to distinguish strategy from luck.\n    \n    Based on: t-statistic = Sharpe * sqrt(N / periods_per_year)\n    We need t > z_alpha for significance.\n    \"\"\"\n    from scipy import stats\n    \n    z = stats.norm.ppf(confidence)\n    \n    #
